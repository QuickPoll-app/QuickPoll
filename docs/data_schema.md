# QuickPoll Data Schema Documentation

## Overview

This document describes the database schema used in the **QuickPoll platform** and the analytics datasets produced by the **Data Engineering ETL pipeline**.

The system is composed of two layers:

1. **Transactional Database (Application Layer)**
2. **Analytics Layer (ETL Outputs for Dashboard Endpoints)**

---

# 1. Transactional Database Schema

These tables support the core QuickPoll application.

---

## users

Stores registered platform users.

| Column | Type | Description |
|------|------|-------------|
| id | UUID | Unique identifier for each user |
| full_name | TEXT | User full name |
| email | TEXT | User email address |
| created_at | TIMESTAMP | Account creation timestamp |

---

## polls

Stores polls created by users.

| Column | Type | Description |
|------|------|-------------|
| id | UUID | Unique poll identifier |
| title | TEXT | Poll title |
| description | TEXT | Poll description |
| creator_id | UUID | Reference to the user who created the poll |
| status | TEXT | Poll status |
| multi_select | BOOLEAN | Indicates if multiple options can be selected |
| created_at | TIMESTAMP | Poll creation timestamp |

### Relationship

```

users.id → polls.creator_id

```

---

## poll_options

Stores options available for each poll.

| Column | Type | Description |
|------|------|-------------|
| id | UUID | Option identifier |
| poll_id | UUID | Poll associated with this option |
| option_text | TEXT | Option description |

### Relationship

```

polls.id → poll_options.poll_id

```

---

## votes

Stores user voting records.

| Column | Type | Description |
|------|------|-------------|
| id | UUID | Vote identifier |
| poll_id | UUID | Poll that was voted on |
| option_id | UUID | Selected option |
| user_id | UUID | User who cast the vote |
| created_at | TIMESTAMP | Vote timestamp |

### Relationships

```

polls.id → votes.poll_id
poll_options.id → votes.option_id
users.id → votes.user_id

```

---

# 2. Analytics Schema (ETL Output)

These tables are generated by the **analytics ETL pipeline** implemented in:

```

data-engineering/etl_pipeline.py

```

They are used for reporting and dashboard endpoints.

---

## analytics_poll_summary

Poll-level aggregated metrics.

| Column | Description |
|------|-------------|
| id | Poll identifier |
| title | Poll title |
| creator_name | Name of poll creator |
| total_votes | Total number of votes received |
| created_at | Poll creation timestamp |
| etl_run_at | ETL execution timestamp |

---

## analytics_vote_trends

Daily voting activity used for time-series dashboards.

| Column | Description |
|------|-------------|
| vote_date | Date of voting activity |
| votes_per_day | Number of votes recorded on that date |
| etl_run_at | Timestamp of ETL pipeline execution |

---

## analytics_user_participation

User engagement metrics.

| Column | Description |
|------|-------------|
| voter_name | User name |
| total_votes_cast | Number of votes cast by the user |
| participation_rate | Share of votes cast relative to total votes |
| etl_run_at | ETL execution timestamp |

---

# 3. Dashboard Dataset Views

These SQL views expose analytics datasets used by **dashboard endpoints and APIs**.

| View | Description |
|------|-------------|
| dashboard_poll_summary | Poll performance analytics |
| dashboard_vote_trends | Voting activity trends |
| dashboard_user_participation | User participation analytics |

These views are defined in:

```

data-engineering/dashboard_views.sql

```

---

# 4. Data Pipeline Architecture

The analytics datasets are produced using the following pipeline:

```

Application Database
         ↓
ETL Pipeline (Python)
         ↓
Analytics Tables
         ↓  
Dashboard SQL Views
         ↓
Dashboard API Endpoints

```

---

# 5. ETL Pipeline Metadata

Pipeline executions are tracked using the metadata table:

```

etl_pipeline_runs

```

This table records:

- pipeline name
- run start time
- run finish time
- rows processed
- execution duration
- pipeline status
